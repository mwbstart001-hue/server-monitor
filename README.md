# Server Monitor - 服务器资源监控脚本

一个轻量级、跨平台的服务器资源监控 Shell 脚本，用于实时采集服务器 CPU 使用率和内存使用情况，并进行统计分析。

---

## 功能特性

- **实时数据采集**：采集 CPU 使用率、总内存、已用内存、可用内存
- **连续采样统计**：连续采集 5 次（可配置），计算平均值、最小值、最大值和波动范围
- **跨平台支持**：同时支持 macOS 和 Linux 系统
- **零依赖**：仅使用标准 Shell 命令，无需安装任何第三方工具
- **健壮的错误处理**：完善的边界情况处理和错误恢复机制

---

## 快速开始

### 环境要求

- macOS 10.12+ 或 Linux (CentOS, Ubuntu, Debian, Alpine 等)
- Bash 3.2+
- 标准系统命令：`awk`, `grep`, `cut`, `sleep`, `hostname`, `top`, `sysctl` (macOS), `vm_stat` (macOS), `free` (Linux)

### 安装使用

1. **下载脚本**

```bash
# 直接下载
curl -O https://raw.githubusercontent.com/mwbstart001-hue/server-monitor/main/server_monitor.sh

# 或者克隆仓库
git clone https://github.com/mwbstart001-hue/server-monitor.git
cd server-monitor
```

2. **添加执行权限**

```bash
chmod +x server_monitor.sh
```

3. **运行脚本**

```bash
./server_monitor.sh
```

## 配置说明

脚本开头的常量可以根据需要修改：

```bash
SAMPLE_COUNT=5    # 采集次数，默认 5 次
INTERVAL=1        # 采集间隔（秒），默认 1 秒
```

## 技术实现

### 平台适配

**macOS**:
- CPU 使用率：`top -l 2 -n 0` 解析 `CPU usage` 行
- 内存信息：`sysctl -n hw.memsize` 获取总内存，`vm_stat` 获取页面使用情况

**Linux**:
- CPU 使用率：`top -bn2 -d0.1` 解析 `Cpu(s)` 行
- 内存信息：`free -m` 获取内存使用情况

### 核心函数

| 函数名 | 功能 |
|--------|------|
| `is_valid_number()` | 验证数值格式有效性 |
| `is_greater_than_zero()` | 检查数值是否大于 0 |
| `compare_numbers()` | 使用 awk 进行数值比较 |
| `safe_divide()` | 安全除法（除零保护） |
| `calc_average()` | 计算数组平均值 |
| `calc_min()` | 计算数组最小值 |
| `calc_max()` | 计算数组最大值 |
| `calc_range()` | 计算波动范围（最大值 - 最小值） |
| `calc_percent()` | 计算百分比（除零保护） |
| `check_command()` | 检查系统命令是否可用 |

## 健壮性设计

### 已修复的风险问题

在开发过程中，我们发现并修复了以下潜在风险：

| 风险等级 | 问题描述 | 修复方案 |
|---------|---------|---------|
| 🔴 高 | 依赖 `bc` 命令 | 完全用 `awk` 替代 `bc`，`awk` 几乎所有 Unix/Linux 系统都默认安装 |
| 🔴 高 | 除零风险 | 添加 `is_greater_than_zero()` 函数检查除数有效性，`safe_divide()` 函数在除零时返回默认值 |
| 🔴 高 | 静默失败模式 | 移除所有 `2>/dev/null`，改为捕获命令退出码；添加命令前置检查；失败时输出警告信息 |
| 🟡 中 | 空数组处理 | 所有统计函数都会检查有效数据数量；无有效数据时返回默认值 `0.00` |
| 🟡 中 | 命令输出格式依赖 | 保留原有解析逻辑，添加 `is_valid_number()` 验证解析结果；解析失败时标记为 "N/A" |
| 🟡 中 | 整数溢出风险 | 大内存计算使用 `awk` 处理（支持任意精度浮点运算），避免 Shell 整数运算溢出 |
| 🟡 中 | `printf` 空值处理 | 添加 `format_number()` 函数，空值或无效值显示为 "N/A" 而非报错 |

### 错误处理机制

1. **前置命令检查**：脚本开始时验证所有必要命令是否可用
2. **数据有效性验证**：每次采集后验证数据格式
3. **采集成功率统计**：记录成功和失败的采样次数
4. **优雅降级**：部分失败时，使用有效数据进行统计
5. **有意义的退出码**：
   - `0`：全部成功
   - `1`：前置检查失败 或 全部采集失败
   - `2`：部分采集失败

---

## 退出码说明

| 退出码 | 含义 |
|--------|------|
| 0 | 执行成功，所有采样都成功 |
| 1 | 执行失败，可能是缺少系统命令或所有采样都失败 |
| 2 | 部分采样失败，但至少有一个采样成功 |

## 常见问题

### Q1: 在 Alpine Linux 上可以运行吗？

**A**: 可以。脚本使用 `awk` 而非 `bc`，Alpine 默认安装了 `awk`（busybox 版本完全兼容）。

### Q2: 为什么需要采集两次 `top` 命令？

**A**: `top` 命令第一次输出的是系统启动以来的平均值，第二次输出才是当前的实时值。因此使用 `top -l 2`（macOS）或 `top -bn2`（Linux）获取准确的实时数据。

### Q3: 如何修改采集次数和间隔？

**A**: 编辑脚本开头的常量：
```bash
SAMPLE_COUNT=10   # 改为采集 10 次
INTERVAL=2         # 改为间隔 2 秒
```

### Q4: 脚本会消耗很多系统资源吗？

**A**: 不会。脚本非常轻量，仅使用标准系统命令，每次执行时间约为 `SAMPLE_COUNT * INTERVAL` 秒，对系统影响可以忽略。
